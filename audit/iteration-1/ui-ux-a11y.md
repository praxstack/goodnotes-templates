## The preview gallery (`preview-server.ts` ¬ß71-139)

### What's there

- Gallery renders `categories ‚Üí files` in a `.grid` of `.card` elements.
- Uses system-default `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`. No Google Fonts fetch ‚Äî good.
- Supports lazy image loading (`loading="lazy"`).
- Uses semantic `<h1>`, `<h2>`, `<p>` ‚Äî passable landmark structure.

## The generated PDF surface

### Applicable WCAG criteria (static document view)

WCAG applies to "web content" ‚Äî PDF accessibility is governed by PDF/UA-1 (ISO 14289-1) and has a distinct toolchain. The CODEX-AUDIT-standard WCAG 2.2 SCs most relevant to static PDFs:

| SC | Applicability | Status |
|---|---|---|
| SC 1.1.1 (Text alternatives) | Decorative SVG stickers have no alt; functional stickers (`checkbox`, `priority-marker` P1/P2/P3) rely on color + text | PNG stickers are just `<img alt="filename">` if embedded at render time; in GoodNotes they don't get alt at all. Not remediable at this layer. |
| SC 1.4.3 (Contrast) | Every template defines CSS variables for text/background. Not measured here. | **NOT MEASURED** ‚Äî Phase 8 tool gap |
| SC 1.4.12 (Text spacing) | Templates use fixed CSS; not resizable in GoodNotes | Acceptable (static artifact) |
| SC 2.4.5 (Multiple ways) | Hyperlinked tabs + bookmarks both present per `FR-02.3` + `pdf-postprocess.ts` | Passes |
| SC 2.5.8 (Target Size ‚Äî min 24√ó24 CSS px) | Tap targets enforced ‚â•44√ó44 pt in `pdf-postprocess.ts:88` | Passes (44 pt > 24 CSS px at any DPI) |
| SC 3.3.2 (Labels or instructions) | Form fields? Templates have no form fields ‚Äî they are paper. | N/A |
| PDF/UA tags | None generated. `pdf-lib` doesn't emit tag tree by default. | Gap ‚Äî PDFs are untagged ‚áí screen-reader navigation is order-only, not structural. |

### Claim reconciliation

- `PROBLEM_STATEMENT.md NFR-06.1` (WCAG AA 4.5:1 text contrast) ‚Äî **NOT MEASURED in this iteration**. Logged as FIND-0018. Remediation path: (a) parse all theme CSS + dark.css variable pairs; (b) compute WCAG contrast ratio; (c) fail if <4.5:1 for normal text or <3:1 for large text.
- `PROBLEM_STATEMENT.md NFR-06.2` ("PDFs SHOULD include basic metadata (title, author, subject) for screen readers") ‚Äî Partial support via `pdf-postprocess.ts addMetadata`. Usage ad-hoc in scripts; default PDFs generated by the CLI `render` command do **NOT** populate metadata. Minor finding (not promoted).
- `PROBLEM_STATEMENT.md FR-04.2` ("Every light theme MUST have a corresponding dark mode variant") ‚Äî **PARTIAL.** 11 of 32 templates have `.dark.css`; 8 themes all have light+dark pairs. Captured as FIND-0018.

## Summary

Neither surface has a user-facing WCAG blocker today. The measurable gaps (contrast audit, PDF tags, dark-mode parity) are enhancement targets; the current state does not regress usability. Phase-8 findings are concentrated in FIND-0018 above.
