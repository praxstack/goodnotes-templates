## Problem Statement

There is no **fully-OSS, fully-free, aesthetically Etsy-grade** system for
generating hyperlinked GoodNotes / Notability / Noteshelf / CollaNote templates,
stickers, and notebook covers. The one real competitor
(`kudrykv/latex-yearly-planner`, 1.5k⭐, MIT) is LaTeX-based, YAML-hell to
configure, e-ink-focused, and ships no stickers, no themes, no modern DX.
Everything else is Etsy paywalls, Patreon gates, or Notion-freemium funnels.

Meanwhile, the shadcn-registry pattern is now the dev community's default
mental model for "copy-paste asset libraries." A 2026-current, MIT-licensed
TypeScript/HTML-CSS implementation of "shadcn for GoodNotes" does not exist.

The repo already has the bones — a self-contained HTML renderer, 60
skeuomorphic stickers, a monthly release bundler, schema-v2 profile data,
UTC-safe date math, and a 143-passing test suite. Approach B packages that
pile of foundation into the shape the dev community expects: `npx
goodnotes-templates add <pack>`.

## What Makes This Cool

**The "whoa" is a single command producing a $25-Etsy-quality PDF.**

```bash
npx goodnotes-templates init
npx goodnotes-templates add prax-journal
npx goodnotes-templates build --month 2026-07
open "output/The Praxis Ledger — July 2026.pdf"
```

Three beats:
1. The dev pastes a familiar `npx` incantation — zero friction.
2. They open the PDF and see torn-paper skeuomorphic stickers, a Fraunces
   nameplate, a bookmark tree that actually navigates on an iPad.
3. They realize this is fully free, fully OSS, and they can fork
   `@goodnotes-templates/prax-journal` into their own registry.

The tweet writes itself: *"@shadcn but for GoodNotes. `npx add
prax-journal` → 130-page hyperlinked journal with 60 stickers. MIT. zero
paywalls. goodnotes-templates.dev"*

## Constraints

**Hard constraints (from locked premises):**
- Fully OSS: MIT for code, CC BY 4.0 for assets. No freemium. No paid tiers.
- Fully offline: `npm install` + `npx` is the entire install surface. No
  mandatory API keys. No hosted services in the critical path.
- GoodNotes-compatible output: PDF 1.7 · sRGB · embedded fonts · 44×44pt
  minimum tap targets · internal hyperlinks only.
- Engineering invariants inherited from v5.3 audit (see AGENTS.md + the
  7 invariants in CONTRIBUTING.md): self-contained HTML · theme-via-override
  · UTC-only date math · XML-safe SVG · size caps.

**Soft constraints:**
- 6-week timeline for v1.
- 7 anchor packs on launch day covering every asset kind in the user's brief
  (templates, stickers, tapes, notebooks, covers) — 1 safe cut allowed.
- One 60-second launch video.
- One-page marketing site at `goodnotes-templates.dev`.

## Premises (all approved by user on 2026-04-30)

1. **Steal `jsrepo` + shadcn's `registry.json` schema wholesale.** Both are
   MIT, both handle non-React payloads, both have extensible `meta` objects.
   Define GoodNotes templates as custom `meta`, reuse the whole toolchain.
2. **The moat is aesthetic quality, not feature count.** The 1,500-star
   LaTeX competitor exists; it's ugly. Every Etsy template is pretty but
   closed. The wedge is "OSS that looks like a $25 Etsy product."
3. **Ship phase D (CLI registry) first; the browser gallery (phase B) waits
   for star signal.** Staggered launch is still "both" — just with
   signal-driven timing.
4. **Phase D's `npx` flow is `init → add → build`**, not
   generate-everything-at-once. Matches shadcn's ergonomics exactly.
5. **Stay 100% offline by default.** No mandatory API keys, no hosted
   services for phase D. No freemium trap.
6. **AI-composer is OUT for phase D**, deferred to a future phase C.
   Requires user-provided API keys = breaks P5.
7. **Distribution:** npm publish (`@goodnotes-templates/core` + per-pack
   sub-packages) + GitHub Releases + one-page marketing site at
   `goodnotes-templates.dev`.

## Landscape Evidence (from Phase 2.75 search)

Three parallel subagents scanned the OSS landscape on 2026-04-30. Key
findings, fully-OSS filter applied (no freemium, no open-core with paid tier):

### The gap is real.
- **No fully-OSS project generates hyperlinked GoodNotes-optimized PDFs with
  stickers, themes, and a modern TS/HTML-CSS toolchain.** Every existing
  project is a partial solution — usually one person's personal planner
  released without sustained engineering investment.
- The one active competitor, `kudrykv/latex-yearly-planner` (1,496⭐, MIT,
  Go + LaTeX), is the 800lb gorilla. E-ink/Supernote/ReMarkable focused.
  Hyperlinks work. But: YAML-hell config, no iPad/GoodNotes optimization,
  no stickers, no themes, no modern DX.

### Prior art to steal.
- **`jsrepo` (jsrepojs/jsrepo, MIT, 624⭐)** — a generalized
  re-implementation of the shadcn registry toolchain. Explicitly advertises
  "distribute your code in any language anywhere." Read end-to-end before
  designing anything.
- **shadcn's `registry.json` / `registry-item.json` schemas** — publicly
  hosted at `https://ui.shadcn.com/schema/`. Extensible `meta` object
  means GoodNotes-specific metadata (paper size, sticker dims, hyperlink
  tree) can ride alongside the standard schema without forking.
- **`pdfme` (pdfme/pdfme, MIT)** — closest prior art for phase B (browser
  composer). Schema-driven JSON templates + in-browser designer. Worth
  exploring when phase B starts, NOT now.

### Eureka moments logged.
- **EU-1** — Everyone assumes "registry" means "React components." The
  shadcn schema is payload-agnostic; the convention is just what the
  community first saw. Re-using the schema for PDFs is a straightforward
  bet, not a stretch.
- **EU-2** — Conventional wisdom: "LaTeX is the serious tool for planner
  generation." First-principles view: typography + CSS are in 2026 just as
  precise as LaTeX for A4 layouts, and modern web fonts + Puppeteer give
  you superior aesthetic control. The LaTeX-is-serious assumption is
  legacy path-dependence, not a technical truth.

## Recommended Approach — Shadcn-exact Mirror

### The deliverable

A 6-week sprint producing:

1. **The `registry.json` schema** at `https://goodnotes-templates.dev/r/registry.json`
   — modeled exactly on shadcn's, with an extended `meta` object carrying
   GoodNotes-specific metadata:
   ```json
   {
     "$schema": "https://goodnotes-templates.dev/r/schema/registry-item.json",
     "name": "prax-journal",
     "type": "registry:asset",
     "meta": {
       "kind": "journal",
       "paper_size": "A4",
       "color_modes": ["default", "dark", "sage", "claude", "cyberpunk"],
       "fonts": ["Fraunces", "Instrument Sans", "JetBrains Mono"],
       "sticker_pack": "prax-journal/stickers",
       "hyperlinked": true,
       "page_templates": ["today", "midday", "reflect", "brain-dump", "weekly", "monthly"]
     },
     "files": [
       { "path": "packs/journals/prax-journal/versions/v5/today.html", "type": "registry:file" },
       ...
     ]
   }
   ```

2. **The CLI**, published as `@goodnotes-templates/core` on npm (MIT).
   Five subcommands:
   - `init` — scaffold a workspace (creates `goodnotes-templates/` dir,
     seeds `shared/fonts/`, writes a minimal `package.json`). No Chromium
     download — that's gated to `build`.
   - `add <pack>` — fetch pack files from the registry, copy into the
     user's workspace, write `meta.json`. User now owns the files —
     no runtime dependency on this repo. **Also copies the
     pre-rendered sample PDF** from GitHub Releases so the user can
     see what they'll get without running `build` once.
   - `list` — print the registry's current packs with metadata (kind,
     paper_size, themes, sticker_pack). Reads from the local registry.json
     cache; refreshable with `--refresh`.
   - `remove <pack>` — delete an added pack and its files cleanly.
   - `build [--pack <name>] [--month YYYY-MM]` — run the renderer
     pipeline on the user's workspace; output to `./output/`. **This is
     the one command that triggers a Puppeteer / Chromium install**
     (see "Install-time UX" below). Shows a progress bar and an
     honest "first build downloads ~200MB Chromium; ~60s on a good
     connection" message before the download starts.

### Install-time UX (addresses the Puppeteer 200MB honesty gap)

The dev-first pitch is "zero friction `npx`" — and we honour that for
`init`, `add`, `list`, `remove`. The Puppeteer/Chromium cost is gated
to `build`, and even there we give users three paths:

1. **Fast path — pre-rendered sample.** `add <pack>` automatically copies
   the most-recent sample PDF from GitHub Releases (e.g. `prax-journal`
   ships with May 2026's 130pp bundle). Zero Chromium. Zero wait. Good
   for users who just want a planner today.
2. **Date-substituted path.** `build --month YYYY-MM --no-render` runs
   only the text-substitution step (`deriveDateFields` + pack-level
   token replacement) and produces a new PDF by splicing the
   pre-rendered template. Ships without Chromium. ~2s. Good for users
   who want the next month's planner but don't want the full build
   toolchain.
3. **Full render path.** `build --month YYYY-MM` is the full pipeline —
   triggers Chromium download on first run if absent. ~60-120s first
   run, ~10s thereafter. Good for users who customize HTML.

The honesty line: **the CLI's front door is zero-friction; the back
door (full render) is honest about its cost.** Most users live at the
front door.

3. **Six anchor packs** live on launch day — one per asset kind the user named
   (*"templates, stickers, tapes, pdf notebooks, covers, etc etc"*):
   - `prax-journal` — the journal flagship · 130pp hyperlinked · templates + stickers
   - `cornell-notes` — the template · 1pp · 8 themes
   - `meeting-notes` — the template · 1pp · 8 themes
   - `habit-tracker` — the template · 1pp · 8 themes
   - `sticker-pack-skeuomorphic` — the stickers · 60 SVG/PNG in 3 dim classes (already rendered)
   - `cover-field-note` — the cover · 1pp · 8 themes · matches the sticker archetype family
   - `blank-notebook-a4` — the notebook · 50/100/200pp blank dot-grid · 8 themes
   - `washi-tapes-skeuomorphic` — the tapes · 12 tileable SVG tapes (DEFERRED to v1.1 if
     Week 1 visual regression time is tight — dropping this is the one safe cut)

   Total: **7 packs** on launch day, 1 deferred. Each pack kind in the user's brief
   ships at v1 or is explicitly deferred with a reason.

4. **One 60-second launch video** — screen-record `npx
   goodnotes-templates init → add prax-journal → build --month 2026-08`,
   open PDF in GoodNotes, scroll through pages showing hyperlinks + stickers
   working. Post to Twitter + Reddit + Hacker News on launch day.

5. **One-page marketing site at `goodnotes-templates.dev`** — Netlify/Vercel
   static page with:
   - `npx` install command (copy button)
   - 3-screenshot gallery (journal · sticker · theme-swap)
   - Link to GitHub · npm · registry.json · the 60s video

### The `registry.json` schema (first cut)

```typescript
// Modeled after https://ui.shadcn.com/schema/registry-item.json
interface GoodNotesRegistryItem {
  $schema: string;                      // pinned to our schema URL
  name: string;                         // e.g. "prax-journal"
  type: "registry:pack" | "registry:sticker-pack" | "registry:theme";
  description: string;                  // for marketing site + gallery
  dependencies?: string[];              // other packs this one depends on
  files: Array<{
    path: string;                       // relative path in user's workspace
    type: "registry:file";
    target?: string;                    // optional rename on copy
  }>;
  meta: {
    kind:
      | "journal" | "planner" | "tracker" | "notes" | "worksheet"
      | "cover" | "sticker-pack" | "tape-pack" | "blank-notebook";
    paper_size: "A4" | "US Letter" | "Supernote" | "Remarkable" | "iPad 10.9";
    color_modes?: string[];
    fonts?: string[];                   // font family names (must be in shared/fonts/)
    sticker_pack?: string;              // path to companion sticker pack
    hyperlinked: boolean;               // does the PDF have internal hyperlinks?
    page_templates?: string[];          // for multi-page packs like prax-journal
    multi_page?: boolean;
  };
}

interface GoodNotesRegistry {
  $schema: string;
  items: GoodNotesRegistryItem[];
}
```
