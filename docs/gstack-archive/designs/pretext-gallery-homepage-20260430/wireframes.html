What these wireframes encode

  • S1 · gallery home — one specimen, not a card grid. Subtraction default. Signature serif italic for "templates" and the masthead. Install command styled as a terminal block. Pack row below fold is a horizontal scroll of real-looking printed pages at slight angles (field notes on a desk), not 3-column tiled icons.
  • S2 · pack detail — full-bleed PDF preview on the left (real content, not lorem-ipsum), tight metadata sidebar on the right. Theme swatches below. Four buttons (download / Codespaces / remix / share) — boring-by-default CTA set, each a different two-way-door.
  • S3 · theme-swap interaction state — captured at the moment the user clicks "caffeine". 240ms transition just completed, toast confirms theme + URL update + render time, preview re-rendered in dark. URL in the sidebar shows the shareable-URL encoding.
  • S4 · Codespaces landing UX — the boot-wait moment (45s target). Not a blank screen. Left pane: a calm "while you wait" tour with progress steps, pedagogy baked in. Right pane: the actual npm ci progress. Makes the 60-second boot commitment feel deliberate instead of slow.

Key design decisions encoded (for Pass 7)

  • Masthead logo is Fraunces italic lowercase "pretext" — signature move.
  • Primary CTA on home is the install command itself, styled as a terminal block. Not "Get started".
  • One accent color per screen. Sage for home + detail. Amber for caffeine. Red for clay horror metric. Never multiple.
  • No cards on home. Pack row uses angled "polaroid" thumbnails to signal "physical artefact" not "SaaS listing".
  • Theme-swap interaction has a toast that confirms render time — developers respect latency numbers.
  • Codespaces landing treats the 45s wait as a product moment — the tour reduces perceived wait.
