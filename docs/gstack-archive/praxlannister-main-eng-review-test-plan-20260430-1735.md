## Affected Pages/Routes
- `/` (gallery home) — pack grid by kind, hero CTAs (install CLI, browse), animated SVG hero
- `/p/[slug]` (pack detail) — preview, theme-swap (E2), download PDF (E3), copy install cmd
- `/p/[slug]?theme=sage` (shareable URL · E1) — deep-link into pack with theme pre-applied
- `/codespaces-open` — GitHub Codespaces handoff (devcontainer boots, runs `add prax-journal`)

## Key Interactions to Verify
- Click pack card on gallery home → navigate to `/p/[slug]` (smooth, <200ms nav)
- Click theme swatch on pack detail → preview re-renders in <300ms (E2)
- Click "Download PDF" button → PDF file appears in Downloads folder (E3 · pdf-lib in browser)
- Click "Open in Codespaces" badge → GitHub opens Codespaces; first boot < 60s; then `npx … build` succeeds
- CLI `init` in empty dir → workspace scaffold · `init` in non-empty without `--force` → clear error
- CLI `add prax-journal` → files copied · meta.json written · pre-rendered sample PDF from GitHub Releases downloaded
- CLI `add` when registry unreachable → falls back to raw.githubusercontent.com/... mirror with "using cached" message
- CLI `build --no-render --month 2026-07` → re-renders in < 2s without Chromium
- CLI `build --month 2026-07` with Chromium missing → honest error + points to `--no-render`
- CLI `remix prax-journal --as test` → new pack dir created · slug renamed · registry.json validates

## Edge Cases
- Theme swap clicked 5 times in 200ms → debounce + cancel-in-flight, no flicker
- E3 download on Safari mobile with 130pp PDF → catches `PDFLibOOMError` → falls back to pre-built PDF link
- `--month 2025-01` (past date) → explicit range guard, clear message: "plans are for future months"
- `add prax-journal` when pack already in workspace → shows diff, prompts "overwrite / skip / abort"
- `add` when registry.json is mid-update → version-pin at session start, no inconsistency
- `remix` when source pack has broken registry entry → validation errors surface
- User-provided month like `'July; rm -rf ~'` → rejected by `/^\d{4}-\d{2}$/` regex (already exists in profile.ts)
- Malicious pack with unicode homograph in `meta.description` → HTML-escaped in gallery render
