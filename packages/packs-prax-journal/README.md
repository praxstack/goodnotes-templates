A personal ADHD + depression journal in the **Warm Analog Editorial** aesthetic, designed for GoodNotes on iPad, authored as HTML and generated to bookmarked A4 PDF.

| | |
|---|---|
| **For** | Prax. Specifically. One user, one body, one set of therapists. |
| **Format** | A4 portrait PDF · bookmarked · base64-inlined fonts. Drag-in stickers (PNG + SVG). |
| **Delivery** | Generate on Mac → AirDrop to iPad → import as GoodNotes notebook. |
| **Release name** | _The Praxis Ledger — &lt;Month YYYY&gt;_ |
| **Tagline** | _A Monthly Daybook for the Unquiet Mind_ |

## The daily spread (v5 · current)

Four A4 pages per day, opened in sequence across the day:

```
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│   TODAY         │ → │   MIDDAY        │ → │   REFLECT       │ → │   BRAIN DUMP    │
│ · Morning       │   │ · 2-second      │   │ · Evening       │   │ · Free canvas   │
│   commit        │   │   pause         │   │   process       │   │ · Sticker       │
│ · One thing     │   │ · Body scan     │   │ · Triad         │   │   watermarks    │
│ · 3 ifs         │   │ · Jar visual    │   │ · Named         │   │                 │
│ · Rx + meds     │   │ · Re-anchor     │   │   patterns      │   │                 │
│ · Cigs baseline │   │                 │   │ · Rx card       │   │                 │
│ · Chest-kg      │   │                 │   │                 │   │                 │
└─────────────────┘   └─────────────────┘   └─────────────────┘   └─────────────────┘
```

The daily spread is the **behavioral system**: morning commit → midday re-anchor → evening process → free writing. Spliced in every Sunday: a weekly review. Every month-end: the monthly review page. Every quarter: the quarterly page.

## Monthly release bundle

One command assembles the self-contained release folder:

```bash
# 1 · Render the month's bookmarked PDF
npx tsx scripts/generate-journal.ts \
  --from 2026-06-01 --to 2026-06-30 \
  --out output/journal-jun-2026.pdf

# 2 · Bundle into "The Praxis Ledger — <Month YYYY>/"
npx tsx scripts/bundle-release.ts \
  --pdf output/journal-jun-2026.pdf \
  --month "June 2026" \
  --from 2026-06-01 --to 2026-06-30
```

Produces `output/The Praxis Ledger — June 2026/`:

| Path | What |
|---|---|
| `The Praxis Ledger — June 2026.pdf` | Bookmarked A4 PDF (≈135 pp · 39 bookmarks for a full month) |
| `The Praxis Ledger — June 2026.html` | Standalone HTML — each page renders as a framed A4 card, base64 fonts deduped, ≤ 2 MB |
| `README.md` | Bundle-level readme |
| `assets/fonts/` | Fraunces · Instrument Sans · JetBrains Mono (self-hosted) |
| `assets/css/` | `base.css` + 14 theme files |
| `assets/source-html/` | v5 per-section templates (`today.html`, `midday.html`, `reflect.html`, `brain-dump.html`, `weekly.html`, `monthly.html`, `quarterly.html`, `design-system.html`) |
| `sticker-pack/pngs/` | `all/` (60) · `compact-400x600/` · `standard-600x600/` · `expanded-800x600/` |
| `sticker-pack/svgs/` | 60 flat SVG sources |
| `sticker-pack/README.md` | Pack-level readme |

**Real file copies — no symlinks.** The folder zips cleanly, AirDrops, and drags into GoodNotes as a self-contained artifact. Lives under `output/` (gitignored, ≈760 MB/month).

## Placeholders (safe, whitelisted)

`substituteProfile()` only interpolates keys it knows. Anything not on the list is left untouched.

| Family | Example keys | Source |
|---|---|---|
| `DR_*` | `DR_NAME`, `DR_SUPERSCRIPT_CREDENTIALS`, `DR_REG_NO`, `DR_FOLLOWUP_DAYS` | `profile.local.json` |
| `RX_1_*` … `RX_8_*` | `RX_1_NAME`, `RX_1_DOSE`, `RX_1_SCHEDULE` | `profile.local.json` |
| `DAY_*` | `DAY_DATE`, `DAY_WEEKDAY`, `DAY_OF_YEAR`, `DAYS_IN_YEAR` | Derived at render time from `PageSpec.date` |

`DAY_*` math is UTC-only (no DST drift). Leap years are handled (`DAYS_IN_YEAR` returns 366 for Feb-29 containing years). Covered by 14 unit tests in `tests/unit/prax-journal-renderer.test.ts`.

## Month-end loop

1. Live in the PDF for ~30 days.
2. At month-end, export the filled notebook from GoodNotes as PDF, AirDrop back to Mac.
3. AI reads and aggregates — see `docs/monthly-generation-playbook.md` (pending).
4. AI generates `output/monthly-review-<yyyy-mm>.pdf` — one page, pre-filled with real data from the journal run.
5. Import back to GoodNotes, reflect, decide next-month iteration scope.

See [`/docs/plan-ceo-review-v3-refined.md`](../../../docs/plan-ceo-review-v3-refined.md) for the full loop rationale.
